# Cross-Project Ideas Pool

Pool переносимых идей между проектами. Идея рождается в одном проекте, оформляется здесь — потом предлагается в других подходящих проектах.

## Структура

- [`INDEX.md`](INDEX.md) — индекс с таблицей статусов по каждому проекту
- [`ideas/NNN-<slug>.md`](ideas/) — отдельный файл на каждую идею

## Workflow

### Создание новой идеи

Когда в работе над проектом X рождается переносимая фишка:

1. **Проверь дубль** — есть ли уже подобная в `INDEX.md`?
2. **Создай файл** `ideas/NNN-<slug>.md` (следующий номер по порядку):
   ```markdown
   # NNN — <Заголовок>

   **Status (overall):** ⚠️ proposed
   **Born in:** <ProjectX>, <дата>
   **Born from:** <одна строка о причине>

   ## Проблема
   Что это решает?

   ## Решение
   Как это работает на практике?

   ## Применимость
   ### applicable_when
   - <условия применимости>
   ### not_applicable_when
   - <анти-условия>

   ## Implemented in / not_applicable_for
   | Проект | Статус | Дата | Заметка |
   |---|---|---|---|
   | <Project> | ✅ | <date> | <pilot/expanded/declined> |

   ## Как переносить в новый проект
   <step-by-step>

   ## Возможные расширения (на потом)
   ```
3. **Обнови `INDEX.md`** — добавь строку с `❓` для всех проектов кроме источника (там `✅ дата`).

### Предложение идеи в другом проекте

При работе в проекте Y Claude в `/start` проверяет `INDEX.md`. Если есть идея со статусом `⚠️` или `❓` для Y — упоминает одной строкой: «Кстати, в `<X>` мы внедрили `<идея>` — может и сюда применить? Подходит потому что `<причина>`». **Не настаивает** — это предложение.

### Приём / отклонение

- **Применено** → файл идеи обновить (`Implemented in` строка с `✅ дата`), `INDEX.md` строку поменять на `✅ дата`.
- **Не подходит** → файл идеи: добавить в раздел «not_applicable_for» строку с причиной, `INDEX.md` поменять на `⛔`.

## Когда писать новую идею

- ✅ Пользователь явно говорит «эту фичу предлагай в других проектах»
- ✅ Сам замечаешь переносимый паттерн: решает класс проблем, не баг; уже задеплоено и работает; не привязан жёстко к стеку
- ❌ Сырые идеи «давайте попробуем» — сначала в `inbox/`, после доводки до решения — сюда
- ❌ Конкретные правки кода — не переносимы, это per-project work
- ❌ Дубли — сначала проверь `INDEX.md`

## Принцип «с умом»

Лучше пропустить идею, чем впихнуть костыль. Pool — для **проверенных** паттернов, не для гипотез.
