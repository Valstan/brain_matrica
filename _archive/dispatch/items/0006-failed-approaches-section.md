## Контекст

Из research-обхода практик handoff-документов (2025-26): паттерн **«секция Failed approaches»** — фиксация что пробовали и не сработало. В commit'ах обычно отражены **успешные** решения, а тупики — нет. Без отдельного места для уроков они теряются и следующая сессия может повторить.

Пользователь зафиксировал позицию (2026-05-22):
- **SESSION_HANDOFF.md обязательно везде** — это место для нитки и failed approaches
- **DEV_HISTORY/DEVELOPMENT_LOG** — оценить нужны ли, многое уже в git коммитах
- Если кто-то упраздняет DEV_HISTORY — **secция Failed approaches в SESSION_HANDOFF становится единственным местом для уроков**

## Уровень обязательности

**Полезно везде, не критично нигде.** Уроки могут жить и без отдельной секции — в commit messages (`fix: попробовал X, не сработало → Y`), в ADR «Alternatives considered», в комментариях коммитов. Failed approaches секция — это **удобное сводное место**, не единственное.

Пользователь зафиксировал (2026-05-22): «парадигма меняется, нейросети мощные, в коммитах уже всё записано основное». В этом контексте — отдельная секция полезна для **длинных ниток** (рефакторинги на N сессий), не для атомарных PR.

## Предложение

Добавить в `SESSION_HANDOFF.md` (после применения #0004 — везде есть) секцию **«Failed approaches (этой нитки)»**:

```markdown
## Failed approaches (этой нитки)

- **<подход X>** — попробовали в коммите [<hash>](<github-url>). Не сработало потому что <причина>. **Не повторяй** без новой инфы.
- **<подход Y>** — рассматривали, отклонили на стадии обсуждения по причине <...>.
```

Дисциплина:
- `/close_session` — при закрытии сессии: если было «попробовали X, не получилось» — записать.
- `/start` — если секция не пуста, выводить в шапку «📛 Известные тупики».

## Возможные «нет»

- Если в проекте уже есть **достаточно мест для уроков** (CLAUDE.md «Технические уроки», ADR «Alternatives considered», commit messages с подробностями) — отдельная секция может дублировать.
- Если **рефакторинги короткие** (одна сессия — одна задача) — секция почти всегда пустая.
- Если пользователь редко повторяет тупики — паттерн избыточен.
