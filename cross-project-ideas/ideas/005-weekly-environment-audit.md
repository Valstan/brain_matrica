# Идея №005 — Weekly environment audit всех проектов

**Date:** 2026-05-22
**Status:** Proposed
**Возникло из:** brain должна помнить чем «живёт» каждый проект чтобы помочь развернуть его на новом компе. Эта информация дрейфует — новые env vars, смена runtime, изменение БД. Если не освежать — устаревает за месяцы.

## Идея

Раз в неделю (или по триггеру `/start` если прошло > 7 дней с последнего audit) brain прогоняет `project-auditor` по всем проектам реестра и **сверяет** актуальную информацию с тем, что лежит в `projects/<P>.md`:

1. **Стек** — изменился ли `package.json` / `requirements.txt` / `pyproject.toml`? (новые / удалённые крупные deps)
2. **Environment** — появились ли новые env vars в `.env.example`?
3. **Прод/runtime** — изменилось ли что в systemd-юнитах / скриптах деплоя?
4. **Фаза** — последний коммит / активная нитка / нет ли стагнации

При расхождении — пишет письмо в `to-brain/` (как от себя самой, kind=feedback) с предложениями обновить файл проекта.

## Зачем

- Bootstrap нового компа работает только если `projects/<P>.md` актуален. Без regular audit информация деградирует.
- Появление нового пререквизита (например, новый внешний сервис, новый env var) — должно автоматически отражаться в реестре brain.
- Это страховка от «забыли что нужно для проекта X» при возвращении к dormant-проекту через полгода.

## Где может быть применено

- **MatricaRMZ** — ⚠️ нужно: имеется много env vars, ledger-publish, sub-services. Особенно полезно.
- **GONBA** — ⚠️ нужно: версии pnpm критичны (10 vs 11), Next.js версии.
- **setka** — ⚠️ нужно: Python версии 3.11 vs 3.12, токены VK_TOKEN_<NAME>.
- **KARMAN** — ⚠️ нужно: проект отстаёт, environment-секция собрана из README, может быть неточна.

## Реализация (черновик)

- Расширить (или создать новый) skill `/weekly-audit` который обходит все проекты read-only
- Сравнивает текущее содержимое `<P>/package.json` / `<P>/requirements.txt` / `<P>/.env.example` с тем что зафиксировано в `projects/<P>.md`
- Diff кладёт в `to-brain/` каждого проекта как kind=feedback urgency=normal
- В meta-сессии в `/start` поднимается «есть N writes от auditor»
