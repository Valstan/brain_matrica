# brain_matrica — Claude Code instructions

## Что это

Это **meta-репо** управления всеми персональными проектами разработчика @valstan. Здесь нет кода приложений. Здесь:
- Реестр проектов с метаданными
- Pool переносимых идей между проектами
- Tech-radar
- Cross-project ADRs
- Inbox сырых идей

## Language
Все финальные сообщения, summary, объяснения, рекомендации пользователю — **на русском**.
Internal reasoning, commit messages, идентификаторы — English.

## Источники правды

Эти файлы хранят состояние работы между сессиями. Читать в начале каждой новой meta-сессии (это делает `/start`).

- [`docs/SESSION_HANDOFF.md`](docs/SESSION_HANDOFF.md) — **sticky-note последней meta-сессии**: что обсуждали, что следующее, ссылка на план если есть. Заполняется `/close_session`, читается `/start` шагом 0.
- [`projects/INDEX.md`](projects/INDEX.md) — реестр всех проектов с метаданными (стек, статус, прод-URL, repo, владелец, текущая версия). Обновляется когда что-то меняется в проекте на стратегическом уровне (новый прод, смена стека, заморозка).
- [`cross-project-ideas/INDEX.md`](cross-project-ideas/INDEX.md) — индекс pool переносимых идей. Каждая идея — отдельный файл в `ideas/`. Полная инструкция работы — в [README](cross-project-ideas/README.md).
- [`tech-radar/INDEX.md`](tech-radar/INDEX.md) — статусы технологий (в эксплуатации / на радаре / отвергнуто).
- [`adr/INDEX.md`](adr/INDEX.md) — cross-project Architecture Decision Records.
- [`inbox/INDEX.md`](inbox/INDEX.md) — сырые идеи и новости индустрии прежде чем оформить.
- [`docs/plans/`](docs/plans/) — большие планы стратегических изменений (миграция нескольких проектов на новый паттерн, etc).

## Команды управления сессией

- `/start` — онбординг новой meta-сессии: подхватывает SESSION_HANDOFF, синхронизируется с origin, читает реестр и pool, докладывает состояние.
- `/close_session` — закрытие meta-сессии: сохраняет «куда мы шли» в SESSION_HANDOFF, делает commit+push handoff'а отдельным коммитом.

## Как мыслить на мета-уровне

В этом репо Claude должен думать **поверх отдельных проектов**, не «глубоко в коде»:

1. **Стратегические вопросы** — какой проект развивать активно, какой заморозить, какой merge с другим
2. **Tech-radar** — что нового в индустрии стоит попробовать; что в эксплуатации устарело и пора заменить
3. **Pool идей** — какая фишка из проекта X применима в проекте Y; какая идея «отжила» и пора удалить
4. **Cross-project ADRs** — «везде используем X», «уходим от Y»
5. **Reflection** — какой проект в длинной паузе и почему, нет ли блокеров от пользователя

Не лезь в код конкретных проектов из этой сессии — для этого открой соответствующий проект отдельно. Здесь — только их **метаданные и стратегия**.

## Связь с обычными проектами

В каждом code-проекте (MatricaRMZ/CLAUDE.md, GONBA/CLAUDE.md, setka/CLAUDE.md) есть ссылка на этот репо. Когда Claude работает в обычном проекте и хочет проверить кросс-проектные идеи или предложить переносимую фишку — он читает `<абс путь>/brain_matrica/cross-project-ideas/INDEX.md`.

Локальные пути могут отличаться на разных компах. В CLAUDE.md проектов — **относительный паттерн**: `../brain_matrica/` (если оба клона в `~/dev/`) или `../../brain_matrica/` (если в `D:/GitHubReps/`). При несовпадении — fallback на `~/.claude/cross-project-ideas/` (legacy) с предупреждением.

## Code style

Это репо не имеет кода — только markdown. Стиль:
- Краткость важнее красоты
- Списки и таблицы предпочтительнее длинных параграфов
- Каждая идея / решение — отдельный файл (один файл — одна мысль)
- INDEX.md в каждой папке — лёгкая таблица для навигации, не полное содержимое

## Что НЕ делать

- Не дублировать содержимое DEVELOPMENT_LOG / PENDING_FOLLOWUPS из проектов — здесь только стратегия
- Не вести в этом репо tactical work любого проекта — это для другой сессии в правильном репо
- Не делать большие монолитные документы — режь на отдельные файлы (idea-X, adr-X, tool-X)
- Не пытаться автогенерировать реестр или radar из проектов — это куратируемый список, не отчёт

## Bootstrap

Когда читаешь этот CLAUDE.md впервые (новая сессия в brain_matrica) и видишь что `cross-project-ideas/`, `tech-radar/`, `adr/`, `inbox/`, `projects/` пустые или почти пустые — **это нормально**, repo только что создан. Текущая задача описана в `docs/SESSION_HANDOFF.md`. Начни с `/start`.
