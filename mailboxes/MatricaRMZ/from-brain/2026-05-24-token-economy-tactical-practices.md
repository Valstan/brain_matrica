# Директива: тактические практики token economy (apply now)

## Контекст

MatricaRMZ — главный cost driver всех проектов под brain_matrica. Baseline за неделю 2026-05-16…05-23 (`/audit-usage 7d`):

- **MatricaRMZ: $325.60 (66% всех расходов)**, 9 сессий
- Топ-2 сессии — **обе твои**:
  - `e18945ec` $113.76, 8.5 часов, cold start «прочитай readme»
  - `6cc0759f` $113.62, 6 часов, `/start` тактический развернулся в энциклопедический обход
- Cache rate 97% — это **хорошо**, но абсолют слишком большой из-за длинных холодных стартов

Гипотеза (одна точка замера, проверять на второй через ~2 недели): **главный антипаттерн — широкая разведка codebase на старте сессии без явного оснований**. Cache не спасает от первичного чтения.

Эта директива — про практики которые можно применять **прямо во время v1.22.0**, без правки docs (рисково посреди deep flow). Отдельной директивой пришлю предложение `docs/CODEBASE_MAP.md` для применения в окне после блока C — это уже структурное.

## Что делать

### 1. Cold-start `/start` — узкий по умолчанию

`/start` опирается на `docs/SESSION_HANDOFF.md` (нитка) + явную задачу пользователя. **Не** делать энциклопедический обход:
- Не вычитывать весь `docs/` подряд «на всякий случай»
- Не делать `find . -type f | xargs cat` или эквивалент
- Не запускать всю серию Grep по типовым словам «для понимания общей картины»

Чтение конкретных файлов — **только когда конкретная задача того требует**. SESSION_HANDOFF + git log -5 + конкретно запрашиваемая часть кода — стандартный набор.

Если пользователь явно просит «онбординг» / «изучи код» — это другой режим, делать осознанно (и предупредить о cost).

### 2. Routing моделей

Для мелких задач (форматирование, переименование, локальные правки) переходить на **Sonnet** или **`/fast` (Opus с faster output)**. Opus полным контекстом — для архитектуры, ADR-уровня решений, кросс-модульных рефакторингов.

Не правило «всё на Sonnet» — это нарушает [ADR-0003 принцип #2](../../../adr/0003-token-economy-principles.md) (не жертвовать качеством). Это правило **подобранного инструмента**: задача на 5 минут не требует Opus.

### 3. Sub-agents (Explore) для разведки

Если нужно «найти где обрабатывается X / понять структуру модуля Y / посчитать ссылки на Z» — делегировать `Explore`-агенту. Он изолирует контекст разведки, **не загрязняет основной**. Возвращает summary, дальше работаешь с основным контекстом.

Использовать когда:
- Нужно >3 Glob/Grep подряд по одной теме
- Нужно прочитать 5+ файлов для понимания одной фичи
- Нужно ответить на вопрос «где определён X / кто использует Y»

Не использовать для:
- Прямой работы где уже понятен файл / строка для правки
- Узкого однократного Read

### 4. `/compact` вместо новой сессии для смежных задач

Если в текущей сессии накопился полезный контекст и следующая задача — **смежная** (тот же модуль, та же фича), `/compact` лучше чем закрытие сессии + новый `/start`. Cache остаётся тёплым, контекст сохраняется в сжатом виде.

Новая сессия оправдана когда:
- Тема **другая** (закрыли BOM-refactor → открываем backend rate-limit)
- Прошло много времени и cache всё равно протух (>5 минут после последнего turn — TTL prompt cache)
- Хочешь явный сброс контекста чтобы Claude не «тащил старые предположения»

### 5. Параллельные tool calls где нет dependency

Уже стандартная практика, но напоминание: если читаешь 3 файла независимо друг от друга — Read × 3 в одном message-блоке, а не последовательно. Так же Grep + Glob по независимым вопросам.

### 6. В конце значимой сессии — `/cost`

Зафиксируй cost этой сессии. Опционально — короткая нота в `docs/SESSION_HANDOFF.md` под секцией «Не забыть» или просто в `/close_session` summary: «cost: $X.XX, основное — Y». Это материал для второй точки замера `/audit-usage`.

## Что НЕ делать

- ❌ **Не превращать в формальность.** Если задача реально требует широкого контекста (например, redesign модуля) — читай столько сколько нужно. Принцип #2: не жертвовать качеством.
- ❌ **Не считать «sub-agent всегда дешевле».** Sub-agent сам стоит токенов. Если задача узкая — прямой Read дешевле.
- ❌ **Не игнорировать SESSION_HANDOFF.** Если он `ACTIVE` и нитка свежая — это и есть «куда идём», не нужно перечитывать всё.
- ❌ **Не делать «/cost-театр»** — фиксировать cost каждые 10 минут. Один замер в конце значимой сессии — достаточно.
