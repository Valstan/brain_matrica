# Projects Registry

Реестр всех проектов под управлением. Обновлять при значимых событиях (новый прод-URL, смена стека, заморозка/возобновление, смена фазы).

## Колонка «фаза» — как читать

- **deep flow** — внутри многоэтапной нитки, есть детальный план на N релизов, не отвлекать побочными заявками
- **PoC mode** — выбираем подход / готовим proof-of-concept перед большим этапом
- **between threads** — текущая нитка только что закрыта, следующая ещё не выбрана; **окно** для техдолгов и побочных задач
- **paused** — сознательная заморозка (ждём решения / приоритет на другом)
- **dormant** — фактически не разрабатывается > 60 дней без явного решения «пауза»

## Kill-policy (Levels-style)

Чтобы заброшенные проекты не подъедали внимание молча — формальные критерии для пересмотра статуса. Применяются в brain_matrica-сессии (обычно при `/start` или morning briefing).

| Условие | Действие |
|---|---|
| **Нет коммитов 30 дней** | Triage в brain_matrica: проверь причину. Может быть осознанной паузой (тогда `paused` явно, с датой), может быть забытым проектом. |
| **Нет коммитов 60 дней** + не помечен `paused` | Фаза автоматически → `dormant`. Заявка в проект: «состояние пересмотреть — продолжаем, замораживаем или закрываем?» |
| **Нет коммитов 90 дней** + `dormant` | Обсудить **kill**: либо явная `paused` с целью (например, «до Q3 2026»), либо `archived` (репо переходит в архив на GitHub, файл проекта в `projects/_archive/`, строка из активной таблицы удаляется). |
| **Активная нитка в SESSION_HANDOFF не двигается > 30 дней** | Заявка: «нитка X застряла на шаге Y. Разморозить, переформулировать или закрыть?» |
| **Нерассмотренные заявки в `inbox-from-brain/` > 30 дней** | Brain_matrica помечает их `superseded` автоматически. Если все заявки регулярно `superseded` — проект в фактическом `dormant`. |

Не «жёсткое правило» — **триггер для разговора**. Решение всегда твоё, не brain_matrica автоматически архивирует.

## Как заполнять файл проекта

Шаблон в [_template.md](_template.md). Минимум для каждого проекта:

- **Repo URL** — GitHub
- **Local clone** — стандартный путь на компе
- **Стек** — основные технологии (язык, фреймворк, БД, deploy)
- **Прод** — URL, версия, deploy метод
- **Owner / SSH alias** — кто работает, какой SSH host алиас
- **Текущая фаза** — что разрабатывается сейчас (если знаем)
- **Связанные идеи** — ссылки на pool (e.g. идея №001 применена)
- **Связанные ADRs** — ссылки на cross-project ADRs которые касаются этого проекта
