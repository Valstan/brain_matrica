# Inbox

Сырые идеи, новости индустрии, черновики прежде чем оформить в pool / radar / ADR / archive.

## Зачем

Каждая «вычитал интересное» / «подумал на досуге» идея падает сюда — иначе забудется. В brain_matrica-сессии разбираем inbox: что-то оформляем (pool/radar/adr), что-то отвергаем (move to archive).

## Структура

- `YYYY-MM-DD-<slug>.md` — каждая идея отдельный файл (дата = когда прилетело)
- Содержимое — свободное: ссылка на статью + 2-3 предложения «почему интересно», без формальной структуры
- `archive/` — отвергнутые после разбора, чтобы не возвращаться к ним повторно

## Workflow

1. **Записать** — `YYYY-MM-DD-<slug>.md` коротко. Не пиши длинно, это inbox а не дока.
2. **Разобрать** в brain_matrica-сессии:
   - **Применимо к проекту X** → переезжает в pool как идея №NNN
   - **Технология для оценки** → переезжает в tech-radar/tools/
   - **Стратегическое решение** → переезжает в adr/
   - **Не подходит** → move to `archive/` с причиной
3. **Не копить** — пустой inbox = хорошо. Цель — обработать, не хранить.
