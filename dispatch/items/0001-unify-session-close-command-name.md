## Контекст у brain_matrica

Что я видел при формулировании:
- В `MatricaRMZ/.claude/commands/` и `GONBA/.claude/commands/` есть `close_session.md` — команда закрытия сессии.
- В `setka/.claude/commands/` есть `finish.md` — та же роль, другое имя.
- При параллельной работе в нескольких проектах разные имена создают micro-friction: переключаешься в setka — рука по привычке набирает `/close_session`, не работает; в MatricaRMZ — наоборот.

Чего я **не** видел:
- Внутренняя логика каждой команды (могут сильно отличаться по содержанию)
- Возможные внешние ссылки на конкретное имя (документация, чужие memory-записи)
- Привычка пользователя: может одно из имён уже въелось — тогда выбор очевиден

## Предложение

**Согласовать одно имя** для команды закрытия сессии и привести к нему все три проекта. Имя файла в `.claude/commands/` — общее, **содержимое команды остаётся проектно-специфичным** (у MatricaRMZ свой релизный flow, у GONBA — Payload, у setka — DEV_HISTORY).

Кандидаты на имя:
- **`/close_session`** — текущее в MatricaRMZ + GONBA. Выразительно, явно «закрытие сессии». Длинно.
- **`/finish`** — текущее в setka. Короче, парный к `/start`. Привычное английское слово.
- **другое** — например `/end`, `/wrap`.

## Зачем

- Снимает micro-friction при параллельной работе с тремя проектами.
- В brain_matrica-сессии в инструкциях `/close_session` — третий вариант. Унификация — это идея #003 в pool, но сейчас она применена частично (только SESSION_HANDOFF, не имя команды).
- Если в будущем появится глобальный slash-aliases механизм — единое имя проще ввести.

## Возможные «нет»

- Если у проекта в DEV_HISTORY / docs / memory активно фигурирует **точное** старое имя — переименование создаст рассинхрон ссылок. Поиск по `git grep close_session` / `git grep finish` покажет масштаб.
- Если автор-пользователь конкретно **не** путается с разными именами (у каждого проекта своя «дверь»), унификация — устранение проблемы которой нет. Это «было ли это вообще проблемой» вопрос.
- В setka `/finish` парный к `/start`, в MatricaRMZ/GONBA `/close_session` — пара к `/start`-`/reliz`-`/close_session`. Семантически в каждом своя логика; принудительная унификация может «срезать» нюанс.

## Что предлагаю по выбору имени

Brain_matrica **не** ставит конкретное имя в заявку — это решение проекта-плюс-пользователя. Каждая проектная сессия в ответе на заявку:
1. Сообщает «текущее имя у нас X».
2. Если поддерживает унификацию — предлагает своё видение какое имя оставить.
3. Если **против** унификации с обоснованием — отклонить заявку.

Brain_matrica в следующей сессии увидит ответы всех трёх и (если есть консенсус) сформирует заявку #NNNN «переименовать в выбранное».
