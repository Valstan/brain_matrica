Закрытие meta-сессии в brain_matrica. Цель — оставить **explicit pointer** «куда мы шли на мета-уровне», чтобы следующая сессия мгновенно подхватила.

## Когда вызывать

- После значимого обсуждения которое не завершилось решением (продолжим позже)
- После добавления новой идеи в pool / новой записи в tech-radar / нового ADR
- После заполнения / обновления реестра проектов
- **НЕ вызывать** — после короткой консультации без записей; после полностью завершённой задачи (нитки нет, `Status: IDLE`)

## Алгоритм

### 1. Собрать контекст meta-сессии

- `git log --oneline -10` — что коммитили в этой сессии
- `git status --short` — что не закоммичено
- TaskList — текущие задачи если есть
- Прочитай текущий `docs/SESSION_HANDOFF.md` — что было в начале

### 2. Спросить пользователя (если развилка)

- Какая нитка активна для следующей сессии?
- Что считается «следующим шагом»? Если задача завершена — пиши `Status: IDLE`
- Если в этой сессии прилетели новые сырые идеи — они уже в `inbox/`?

### 3. Записать docs/SESSION_HANDOFF.md

Перезаписать целиком. Структура:

```markdown
# brain_matrica — Session Handoff

**Status:** ACTIVE | IDLE
**Updated:** YYYY-MM-DD (Claude session)
**Branch:** main

## Текущая нитка
1-3 предложения. На мета-уровне (не tactical в каком-то проекте).

## Следующий шаг
Конкретно: что обсудить / создать / разобрать в следующей brain_matrica-сессии.

## Контекст
- План (если есть): docs/plans/X.md
- Связанные коммиты: <hashes>
- Inbox: <количество необработанных>
- Pool: <идеи добавленные в этой сессии>

## Открытые вопросы для пользователя
- <если есть>

## Не забыть (low-priority)
```

Если `Status: IDLE` — оставь шапку + пустые секции с `_n/a_`.

### 4. Обновить INDEX.md где нужно

- `cross-project-ideas/INDEX.md` если добавили / переоценили идеи
- `projects/INDEX.md` если обновили реестр
- `tech-radar/INDEX.md` если поменяли статусы технологий
- `adr/INDEX.md` если добавили ADR

### 5. Commit + push

Один маленький коммит:
```
chore(session): handoff — <short summary>
```

Push на origin сразу — handoff доступен с любого компа.

### 6. Финальный отчёт пользователю

- Что закрыто в этой meta-сессии (1 строка)
- Handoff сохранён ✅
- Что подхватит `/start` в следующей сессии

## Что НЕ делать

- Не вызывать после короткой консультации без правок
- Не дублировать tactical work — это для соответствующего проекта
- Не лезть в код-проекты из meta-сессии
- Не делать `git push --force` / `git reset --hard` — handoff commit обычный
